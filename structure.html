What is a doctype and why is it important to include it in every HTML document?

When you add <doctype> at the start of your HTML file you are declaring what version of HTML it has been written in so that the browsers which it is opened in knows what to expect.

What is XML and why is it important to web developers?

XML stands for EXtensible Markup Language and is structured in a similar way to HTML The difference between XML and HTML is the fact that XML allows you to define your own tags instead of using the generic tags you'd normally use in HTML. For example...

<note>

<to>Tove</to>

<from>Jani</from>

<heading>Reminder</heading>

<body>Don't forget me this weekend!</body>

</note>

Notice how instead of using tags such as <h1> I used <heading> instead. XML allows you to create custom tags in your script to simplify the creation of your site.

Describe 4 XML rules?

1. Elements must be properly nested.

2. Elements must have a start and end tag.

3. Element names are case-sensitive.

4. Attribute values must be enclosed in double quotation marks.

What is the difference between a "well-formed" and a valid web document?

A well formed XML document is one where everything in the document is used correctly with no errors this includes opening and closing tags in their correct order and the HTML version is declared on the first line with <doctype> A valid document is where the the document is well-formed but also follows the rules of a specific doctype.
